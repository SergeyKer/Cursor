# Монеты: отмена ошибки на шагах 4–6 (v1)

**Статус:** план + стартовый бонус 10 🪙; UI и списание — следующий этап.  
**Связано:** `lib/lessonCoinForgiveness.ts`, `lib/rewardsState.ts`

---

## 1. Продуктовое правило

| Параметр | Значение |
|----------|----------|
| Цена | **1 🪙** за одну отмену |
| Лимит | **1 раз за проход урока** (`runKey`) |
| Где | **Шаги 4, 5, 6** — translate и sentence_puzzle |
| Когда | Только **после реальной ошибки** на текущем шаге/субшаге |
| Эффект | Ошибка **не ухудшает attempt** при успешном ответе; **combo не восстанавливается** |
| Золото | **Возможно** (одна отмена ≈ одна «спасённая» попытка на unit) |
| Шаги 1–3, 7 | Кнопки отмены **нет** |

### Шаги по типам (каталог 1–4)

| stepNumber | Тип | Отмена за 🪙 |
|------------|-----|--------------|
| 4 | `translate` (полное предложение) | ✅ |
| 5 | `sentence_puzzle` (3 sub) | ✅ (на sub, 1 раз на урок суммарно) |
| 6 | `translate` (финальный экзамен) | ✅ |

## 2. UX: как выглядит опция

### Момент показа

После **неверного ответа** на шаге 4–6, под карточкой статуса (жёлтый тон ошибки) и **над** полем ввода / пазлом:

```
┌─────────────────────────────────────────────┐
│ ⚠️  Почти. Попробуйте ещё раз.              │
│ 💡 После Who нужна форма глагола с -s.      │  ← бесплатная подсказка (как сейчас)
└─────────────────────────────────────────────┘

┌─────────────────────────────────────────────┐
│  Ошибка засчитана для этого задания.        │
│                                             │
│  ┌─────────────────────────────────────┐   │
│  │  🪙 Убрать штраф ошибки — 1 монета  │   │  ← primary outline, жёлто-янтарный акцент
│  └─────────────────────────────────────┘   │
│  Золотая медаль по-прежнему возможна.       │  ← muted, 12px
│  Осталось монет: 10 🪙                      │
└─────────────────────────────────────────────┘

[ поле ввода / пазл — активно, можно отвечать снова ]
```

### После нажатия (подтверждение)

Короткий sheet / inline confirm (особенно `audience: child`):

```
Убрать штраф этой ошибки за 1 🪙?
Серия COMBO не восстановится.

[ Отмена ]  [ Убрать штраф — 1 🪙 ]
```

### После успешной оплаты

- Списать 1 🪙 из `rewardsState.currencies.coins`
- `exerciseErrors` (или puzzle attempts) **−1**, минимум 0
- **Combo остаётся 0** (если уже сбросился)
- Плашка меняется на:

```
✓ Штраф ошибки снят. Попробуйте ещё раз.
```

- Кнопка «Убрать штраф» **скрывается до конца урока** (флаг `forgivenessUsedThisRun`)
- В футере урока (один раз): `Штраф ошибки снят за 1 🪙.`

### Если монет не хватает

```
┌─────────────────────────────────────────────┐
│  🪙 Убрать штраф — 1 монета                 │  ← disabled
│  Не хватает монет (нужно 1, у вас 0).       │
│  [ Пополнить ]  ← позже, v2                 │
└─────────────────────────────────────────────┘
```

### Когда кнопку не показываем

- Уже использовали отмену в этом `runKey`
- Шаг 1–3 или 7
- Статус `checking` / ещё нет ошибки на шаге
- `forgivenessUsedThisRun === true`

## 3. Логика зачёта (движок)

### Translate / fill_text на шагах 4 и 6

1. Пользователь ошибается → `exerciseErrors++`, `combo = 0`
2. Оплата отмены → `exerciseErrors = max(0, exerciseErrors - 1)`, **combo не трогаем**
3. Успех → `attemptIndex: exerciseErrors` при `applySuccessAward` (как без лишней ошибки)

### Sentence puzzle (шаг 5)

- Ошибка на sub → счётчик попыток sub растёт
- Отмена → **−1 к attempts** текущего sub перед следующей отправкой
- Лимит **1 отмена на весь урок** (общий с шагами 4 и 6)

### Прогресс урока (сохранять в runtime)

```ts
type LessonCoinForgivenessRunState = {
  used: boolean
  stepNumber?: number
  at?: string // ISO
}
```

Ключ: `lessonId + runKey`.

## 7. QA

1. Шаг 4: ошибка → кнопка → −1 🪙 → успех с 1-й «зачётной» попытки → золото возможно.
2. Шаг 4: отмена на шаге 4 → на шаге 6 кнопки нет.
3. Combo после ошибки = 0; после отмены всё ещё 0.
4. 0 монет → кнопка disabled.
5. Новый пользователь: 10 🪙 в профиле.
6. Старый пользователь: +10 один раз, повторный load без дубля.
