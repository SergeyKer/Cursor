# Принципы перевода с русского на английский

Краткая методичка для преподавателя и для настройки ИИ. Для пары **like / love** в продукте действует **одна** согласованная политика (см. также [`lib/likeLoveTutorPrompt.ts`](../lib/likeLoveTutorPrompt.ts) и системные промпты чата / [`/api/translate`](../app/api/translate/route.ts) для `ru_to_en`).

## Like и love

**Правило:** выбирай *like* / *love* по **интенсивности** эмоции и смыслу русской фразы, а не по тому, что в русском обязательно стоит слово «люблю».

### Критерий выбора

**Love** (сильная эмоция, ориентир 8–10 из 10):

- Близкие люди: мать, отец, дети, партнёр → *I love my mother*.
- Питомцы **с** эмоциональной связью → *I love my dog* (и это нормально).
- Страстные увлечения → *I love painting!*, если по контексту именно страсть.

**Like** (умеренное предпочтение, ориентир около 3–4 из 10):

- Деятельность / хобби → *I like cooking*, *I like walking*.
- Вещи, еда, погода → *I like coffee*, *I like rain*.
- Питомцы **без** акцента на глубокой привязанности → *I like my dog* (тоже нормально).
- Вежливые мнения → *I like this idea*.

### Алгоритм по русскому оригиналу

1. Смотри **русский** текст:
   - «люблю» + человек / явная страсть → чаще **love**.
   - «люблю» / «нравится» + деятельность / вещь → по умолчанию **like**.
   - «нравится» в нейтральном смысле → **всегда like**.

2. **Питомцы** (собака, кот и т.п.): допустимы **и** *I love my dog*, **и** *I like my dog* — **оба верны**. Не считай ошибкой одну только замену love ↔ like.

3. **Деятельность** (cooking, walking, reading): по умолчанию **like** — *I like cooking*. **Love** уместен, если контекст показывает страсть — например *I LOVE cooking!*

### Примеры

- «Я люблю маму» → *I love my mother* (близкий человек).
- «Я люблю готовить» → *I like cooking* (деятельность по умолчанию).
- «Я люблю собаку» → *I love my dog* **или** *I like my dog* (оба допустимы).
- «Я люблю гулять с собакой» → *I like walking with my dog* (в фокусе деятельность).

### Чего не делать

Не требуй замены только ради like/love, если:

- для питомца ученик выбрал *love*, а эталон *like*, или наоборот;
- для деятельности эталон *I like cooking*, а ученик сказал *I love cooking* **без** контекста страсти в русском — тогда канон обычно остаётся *like*; наоборот не навязывай *love*, если в русском нет сильной эмоции.

**Запрещено:** менять *like* и *love* местами без смысла в **не**-исключённых случаях; требовать *love* при нейтральном «нравится»; требовать *like* там, где по-русски явно сильная привязанность к **человеку** или страсть (не путать с питомцами и деятельностью по правилам выше).

| Принцип | Содержание |
|--------|------------|
| Эталон в упражнении | В режиме перевода в чате канонический английский задаётся эталоном задания; ответ сравнивают с русской фразой и с этим эталоном, с учётом исключений для питомцев. |
| Диалог | Эталон — последняя реплика преподавателя на английском: не требовать *love*, если в вопросе *like*; ответ с *love* при том же смысле можно принять и кратко похвалить. Для питомцев не форсировать love ↔ like. |

## Would like и would love

- **Would like** — вежливое «хочу» (*I'd like to…*).
- **I'd love to** — сильнее по готовности / энтузиазму, чем *I'd like to*.

## Связь с API

По умолчанию [`/api/translate`](../app/api/translate/route.ts) переводит **английский → русский** (кнопка «Перевод» у ответа ассистента).

Для перевода **русский → английский** передайте в теле запроса `"direction": "ru_to_en"` вместе с полем `text` на русском. Подробности — в комментариях к маршруту.
