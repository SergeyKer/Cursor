# План: новый чат при смене настроек (все три режима)

## Зачем

Сейчас при **смене режима** — **Общение** / **Диалог** / **Тренировка перевода** — после закрытия меню чат **сбрасывается** ([`app/page.tsx`](app/page.tsx), `restartChatForNewModeFromMenu`).

При смене **других** настроек в том же режиме история **не** сбрасывается, только баннер «Настройки изменены…».

**Цель:** после закрытия меню, если **режим не менялся**, но изменились релевантные для **текущего** режима поля — тоже вызывать тот же перезапуск (новый «первый» ответ ИИ).

**Уровень (`settings.level`)** в автоперезапуск **не входит** (как договорились): только смена уровня не очищает чат при закрытии меню.

---

## Три режима: что сравнивать

Логика меню: [`MenuSectionPanels.tsx`](components/MenuSectionPanels.tsx) (`menuView === 'aiChat'`).

| Режим | Поля в меню, влияющие на смысл чата | В снимок для перезапуска (без `level`) |
|-------|--------------------------------------|----------------------------------------|
| **Диалог** (`dialogue`) | Тема, время (мультивыбор), «Стиль общения» = ребёнок/взрослый (`audience`). Тип предложений в этом экране **не показывается** (только у перевода), но `sentenceType` хранится в `Settings` — включать в снимок для согласованности с API. | `topic`, `tenses`, `sentenceType`, `audience` |
| **Тренировка перевода** (`translation`) | Как диалог + **тип предложений** (`sentenceType`). | `topic`, `tenses`, `sentenceType`, `audience` |
| **Общение** (`communication`) | Ребёнок/взрослый; **тема и время в этом блоке скрыты** (для API тема фиксируется как свободная). Уровень в меню есть, но **не** триггер перезапуска. | `audience` (и при необходимости позже — отдельно обсудить `communicationInputExpectedLang`, он переключается **в шапке**, не в этом меню) |

Итого по режимам:

- **Диалог** и **Перевод**: один и тот же набор в снимке — `topic`, `tenses`, `sentenceType`, `audience` (без `level`).
- **Общение**: минимально — `audience`; смена только уровня **не** перезапускает.

---

## Поведение для пользователя

1. Открыл меню «Чат с MyEng», активен один из трёх режимов.
2. Поменял настройки из таблицы выше (для общения по сути — ребёнок/взрослый; для диалога/перевода — ещё тема, время, тип предложений у перевода и т.д.).
3. Закрыл меню.
4. **Ожидаемо:** как после смены режима — **чистый лист**, новое первое сообщение от ИИ.

---

## Технически

1. Заменить ref «только режим при открытии» на структуру: **`mode`** + **`settingsSnapshot`** (набор полей зависит от `mode`, см. таблицу).
2. При **закрытии** меню: если `mode` изменился → перезапуск (как сейчас). Иначе вызвать `settingsSnapshotChanged(mode, snap, currentSettings)` и при `true` → **`restartChatForNewModeFromMenu()`**.
3. Вынести сравнение в функцию (например `dialogueLikeSnapshotEqual` для `dialogue`/`translation` с одинаковым набором полей; отдельно `communicationSnapshotEqual` только для `audience`).
4. **Времена**: сравнивать как множество (отсортировать id).
5. Баннер «Настройки изменены…» — проверить, что после автосброса не врёт; при необходимости сузить условие.

---

## Риски

- Один вызов перезапуска: «режим изменился» **или** «снимок настроек изменился» — не дублировать.
- **Общение**: при смене только уровня чат **не** сбрасывается (по требованию про уровень).

---

## Итог

Фича относится к **трём режимам** чата (общение, диалог, перевод), не только к диалогу; набор полей в снимке зависит от режима; **уровень** нигде не используется для этого автоперезапуска.
