# Кнопка Ru↔En в шапке (ожидаемый язык ввода)

## Видимость (обязательно)

- Кнопка показывается **только** при **`settings.mode === 'communication'`** и активном чате (например `dialogStarted`).
- В режимах **«Диалог»** и **«Тренировка перевода»** кнопки **нет** — не рендерить, не занимать место в шапке.

## Поведение (кратко)

- Подпись: при ожидании русского → `Ru→En`, при ожидании английского → `En→Ru`; клик переключает `communicationInputExpectedLang`.
- **Старт:** по умолчанию ожидание **английского** ввода (`communicationInputExpectedLang === 'en'`), кнопка сразу **`En→Ru`**; после перезапуска чата в «Общении» снова **`en`**.
- Поле в `Settings` и в теле `POST /api/chat` используется **только** в ветке `communication`; сервер игнорирует поле при `mode !== 'communication'`.
- Сброс предпочтения при смене режима с «Общение» на другой — вместе с перезапуском чата.

## Синхронизация с фактическим вводом (обязательно)

- В состоянии **ожидания следующего ввода** подпись кнопки должна **соответствовать** актуальному `communicationInputExpectedLang`.
- **После каждой отправки** сообщения пользователем выставлять `communicationInputExpectedLang` по **той же детекции**, что и язык этой строки: есть кириллица → **`ru`**, только латиница (и есть буквы) → **`en`**, пусто/только знаки — не менять или по tie-break (уточнить одно правило в коде).
- Так кнопка **всегда переключается** (обновляется) вместе с тем, какой язык ввода логично ожидать **перед следующей строкой**, и совпадает с диктовкой/заголовком «Чат»/«Chat».
- **Ручной клик** по кнопке временно меняет ожидание до следующей **отправки** пользователем сообщения; после отправки снова применяется авто-синхронизация по тексту (если не решите иной продуктовый вариант — «ручной выбор держится N ходов»).

## Единый алгоритм текста и микрофона (обязательно)

- Алгоритм выбора языка для **текстового ожидания** (кнопка/`communicationInputExpectedLang`) и для **микрофона** (`speechLocaleForCommunication`) должен быть **одинаковым**.
- Один и тот же источник истины: есть кириллица → `ru` / `ru-RU`; есть только латиница → `en` / `en-US`; нет букв → `communicationInputExpectedLang` как tie-break.
- На старте «Общения» (ожидание `en`) микрофон тоже стартует в `en-US`.

## Изоляция от других режимов

- `dialogue` / `translation`: без изменений UI шапки для этой кнопки; логика `speechLocaleForCommunication` и tie-break для общения не вызывается вне `communication`.
